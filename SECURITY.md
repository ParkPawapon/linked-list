# Security Policy

## Scope

This repository is maintained as a production-style sample for local execution and CI validation.

## Supported Versions

The `main` branch is the supported branch.

## Reporting a Vulnerability

1. Do not open a public issue with exploit details.
2. Contact the repository owner directly.
3. Include:
   - reproduction steps
   - impact summary
   - suggested mitigation (if available)

## Secure Development Baseline

- Branch protection on `main`
- Pull request workflow with CI checks
- Dependency vulnerability alerts
- Automated security fix pull requests
- CODEOWNERS review policy
